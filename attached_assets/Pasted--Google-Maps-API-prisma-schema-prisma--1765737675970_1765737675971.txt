任務三：實作商家 API (server/routes/merchant.ts)
Google 地點搜尋 (GET /api/places/search)：
使用 axios 呼叫 https://maps.googleapis.com/maps/api/place/textsearch/json。
API Key: 從 process.env.GOOGLE_MAPS_API_KEY 讀取。
回傳：googlePlaceId, name, address, geometry (lat/lng), types。
注意：不處理圖片。
認領地點 (POST /api/merchant/claim)：
防撞檢查：檢查 googlePlaceId 是否已存在於 MerchantClaim 或 ItineraryCard。
若存在：回傳 409 Conflict，Body: { canDispute: true }。
若不存在：
建立 MerchantClaim (PENDING)。
自動分類：根據 Google types 嘗試對應到我們的 SubCategory (若無法對應則留空，待商家手動選)。
建立 DraftItineraryCard (無圖片)。
提交爭議 (POST /api/merchant/claim/dispute)：
允許針對重複的 ID 建立認領單，但標記 isDispute = true。
請完成上述工作並重啟伺服器。
